Descripción: 
El error del código proporcionado es debida a que el programa se puede ejecutar
sin argumentos. Si ejecuta con un argumento de tipo intero, no se produce el
error. Sin embargo el programa no muestra ningún mensaje de que se debe entrar
un valor numérico. Solamente se lanza la excepción. Para resolver este problema
se implementa un try-catch que trata la excepción y imprime la razón de la
misma.
